The simple differences to relative, absolute, fixed, and static positioning.

Posted Feb 27, 2016.

Many people (myself included) had trouble distinguishing the difference between arrays and hashes when first starting to program. So let's talk a bit about them both.

If you haven't already checked out ruby-doc.org, now would be a good time. It's incredible how much you can learn by reading their documentation on Ruby, and they give great definitions of both arrays and hashes.

Arrays

"Arrays are ordered, integer-indexed collections of any object." While that may seem like a mouth-full, the concept is quite simple. For example, here is an array: new_array = [1, "hello", ["this is an element in a subarray", 5], 3]

An array contains either numbers, strings, or even other arrays (as shown above.) They can be very useful if you need to store or retrieve a large amount of data.

Hashes

"A Hash is a dictionary-like collection of unique keys and their values." Essentially, a hash contains a bunch of key-value pairs and, while it is similar to an array, it has some notable differences.

First, indexing is done via arbitrary keys of any object type. Let's take a quick look at indexing for arrays. For example, if this is your array: new_array = ["hello", "yes", "no"] then new_array(0) = "hello", new_array(1) = "yes", and so forth.

However, hashes are a bit different. Let's say this is your hash: greetings = { :friendly => "Hello, beautiful!", :angry => "Hello, not beautiful!" } Hash indexing is done like this: greetings[friendly] = "Hello, beautiful!"

So when should you use them?

Both hashes and arrays store data and allow you to retrieve that data. However, the way in which you can retrieve and store that data is different for both arrays and hashes.

Arrays have fixed keys, as seen with their numerical indexing, whereas hash keys are much easier to customize. If you still don't quite get it, check out this great answer for when you should use either one.
