Positioning: Revealed!

The simple differences to relative, absolute, fixed, and static positioning.

Posted Feb 27, 2016.

One of the most confusing properties of CSS for beginners is position:. For most it may seem intuitive at first, but there are many values for it, which can make it difficult to remember.

Let's just put it out there - positioning in CSS sucks to work with when starting out. And it also happens to be important since it will help create the layout your envisoning for your website. Since positioning seems to be a combination of confusing and useful, let's try to make it simpler to understand.

The Key Position

The definition from w3schools for position is this:

The position property specifies the type of positioning method used for an element (static, relative, fixed or absolute).

While that pretty much states the obvious, what might not be so obvious is that it actually creates an algorithm for the element that you are targeting, which determines its position. This is pretty cool, so let's go more in-depth into the types of values this property takes to understand how it works under-the-hood.

The Static Value

Static is the default value for the position property. An element with position: static; is not positioned in any special way. This is great news for us, because it means that the element will simply follow the box model. In fact, an element with position: static; is said to not be positioned at all.

The Absolute Value

The absolute value is perhaps the most confusing value for the position property, so let's get it out of the way. An absolutely positioned element will either use its parent element or the HTML body to position itself. (Imagine you are absolutely positioning a h2 tag inside of a div. The h2 position will be relative to the div instead of the entire HTML body tag.)

In order to manipulate an absolutely positioned element, it's actually quite easy. The most useful properties to use with position: absolute; is top:, bottom:, left:, right:, width:, and height:.

The Relative Value

The relative value is almost the same as static, except that you can manipulate its position. As with position: absolute;, you can use the above properties to change a relative positioned element. So if you want to manipulate a static positioned element, make sure you change its position to relative.

The Fixed Value

If an element has position: fixed; it means that the element will be positioned with respect to your device's viewport. Even if you scroll, the element will stay in the same location on your screen. And as with absolute and relative positioning, you can use the above properties to manipulate the element's position.
